# SPARTAN vs IMPROVE — shared-parameter comparison

Generated by [`../spartan_vs_improve_comparison.ipynb`](../spartan_vs_improve_comparison.ipynb).
Re-run that notebook top to bottom to regenerate everything here.

Compares the SPARTAN filter network against the US IMPROVE network using **only
the quantities both networks carry on disk**: EC, OC, optical absorption (fAbs),
iron, and fine PM₂.₅ mass — plus the derived ratios OC/EC, fAbs/EC (≈ MAC), and
Fe/PM₂.₅. Parameters present in only one network (e.g. titanium — SPARTAN has it,
IMPROVE's local export does not) are excluded by design.

The networks share no sites, filters, or dates, so this is a **distributional +
relationship** comparison, not a row-level join.

## Data sources

- SPARTAN: `research/ftir_hips_chem/Filter Data/unified_filter_dataset.pkl`
- IMPROVE: `research/ftir_hips_chem/output/improve_high_fabs_comparison/improve_valid_cleaned.csv`

## EC caveat

SPARTAN stores two non-overlapping filter sets: a *ChemSpec* set (thermal EC/OC,
metals, mass) and an *FTIR/HIPS* set (`EC_ftir`, `HIPS_Fabs`). `HIPS_Fabs` only
co-occurs with `EC_ftir`. Distributions and OC/EC therefore use thermal
`ChemSpec_EC`; the fAbs-vs-EC and fAbs/EC (MAC) analyses use `EC_ftir`. FTIR-vs-
thermal EC is a known sensitivity — read the MAC slope with that in mind.
